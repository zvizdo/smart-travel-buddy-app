# Frontend Gotchas

## Auth Race
`auth.currentUser` is null on initial load. Check `useAuth()` `loading`/`user` before API calls.

## API Client Retry Semantics (`frontend/lib/api.ts`)
Firebase ID tokens are cached for up to an hour by `user.getIdToken()`, so a stale token surfaces as an opaque 401 with no recovery. `request<T>()` detects 401, calls `user.getIdToken(true)` (force refresh), and retries the same request exactly once. A second 401 bubbles up as a thrown error — the client does NOT redirect; callers decide. Every verb (`get/post/patch/delete`) accepts an optional trailing `AbortSignal` that's forwarded to `fetch` and preserved across the retry. Guarded by `frontend/lib/api.test.ts`.

## Firestore `__name__`
Full document path, not just ID. Use `where("id", "==", ...)` or direct lookup.

## Date Handling
`@/lib/dates.ts` with `Intl.DateTimeFormat`. `DateTimePicker` for inputs. Departure-before-arrival validation.

## User Display Names
`formatUserName()` returns "FirstName L." format. Never show raw UIDs.

## Location Tracking
`User.location_tracking_enabled`. Backend rejects pulse if disabled. Disabling deletes all location docs.

## CSS Height Chain
Google Maps needs `html.h-full > body.h-full > container.h-full > map.h-full`. Use `min-h-0` on flex children.

## Overlay Stacking
Glass header `z-20`, DivergenceResolver `bottom-[nav] z-20`, Bottom nav `z-30`.

## Map Camera Stability

`TripMap` initial fit is gated by an internal `hasFittedRef` (reset on `planId` change), not by `useEffect` deps — `initialFitNodes` and `myNodeIds` are memos that rebuild on every Firestore snapshot, so dep-driven effects refit the map on every unrelated update. Post-initial refit triggers ONLY on filter mode toggle ("all" ↔ "mine"), tracked via `prevFilterModeRef` — NOT on content changes to `myNodeIds`. Node/edge mutations (add, delete, edit) must never cause a refit. Refs exposed via React Context (e.g. an old `mapFitted` flag) do not trigger consumer re-renders, so the consumer always reads the first value — gate one-shot side effects inside `TripMap` itself, not via context state.

**Context-aware entry point**: `page.tsx` computes `initialFocalPoint` (priority: user near stop <50km → next upcoming stop → trip root → null=fitBounds) and passes it to `TripMap`. When set, the map does a two-phase fly-in: instant `fitBounds` overview then 500ms animated zoom to the focal point. Trips with >15 stops or completed trips always fall back to `fitBounds`. `savedCamera` (sessionStorage) still takes priority over the focal point for returning users.

## Ref Reads During Render (React Compiler)

`react-hooks/refs` flags `ref.current` in render. Use `useState` lazy initializer for read-once snapshots (e.g. `savedCamera` for `TripMap`). Gotcha: the Compiler silently bails out on other violations, so fixing one can surface latent warnings in the same file.

## Agent Tool Signatures (`agent_tools.py`)

Never add `from __future__ import annotations` — Gemini AFC's runtime arg converter calls `isinstance(value, param.annotation)`, which raises `TypeError` when annotations are PEP 563 strings. Schema generation has a pydantic fallback that hides this; only the invocation path crashes, so symptoms look like "the agent can read but can't mutate." Optional params must be `T | None = None`, not `T = None`, so they pass the SDK's strict schema validator instead of the brittle fallback. Guarded by `backend/tests/test_agent_tools.py`, which runs every tool through the SDK's actual `convert_argument_from_function` and `_parse_schema_from_parameter`.

## Duplicate Edge Prevention
`DAGService._create_edge_if_new()` on all creation paths.

## Implicit Branching

No `branch_id` on edges. Paths derived at runtime from DAG topology + `participant_ids`. Divergence = out-degree>1 or multiple root nodes. `cleanup_stale_participant_ids` (called after `delete_node`) only wipes `participant_ids` when the DAG is truly linear: exactly one root AND no out-degree>1. Multi-root divergence (virtual `__root__` in frontend) is checked via in-degree analysis — nodes with zero incoming edges are roots.

## ID Format
Short typed IDs (`n_`, `e_`, `p_`, `t_`, `a_` prefixes + 8 alphanumeric chars) for agent-friendliness. Generated by `shared/shared/tools/id_gen.py`.
